# 数据传输功能使用指南

## 快速开始

### 第一步：准备数据

在设备A上：
1. 启动TodoList应用
2. 创建一些任务和分类作为测试数据

### 第二步：启动数据共享

1. 点击左侧导航栏左下角的"🔗 数据共享"按钮
2. 确保选中"共享数据"标签页
3. 查看当前数据摘要，确认要共享的内容
4. 点击"启动共享"按钮
5. 记下显示的IP地址和端口（例如：`192.168.1.100:5000`）

### 第三步：接收数据

在设备B上：
1. 确保两台设备在同一局域网内
2. 启动TodoList应用
3. 点击"🔗 数据共享"按钮
4. 切换到"接收数据"标签页
5. 点击"扫描局域网设备"
6. 从列表中选择设备A的IP地址
7. 等待数据加载完成
8. 查看接收到的数据预览
9. **重要**：如果有现有数据，系统会显示警告
10. 点击"确认导入"完成数据同步

## 功能说明

### 共享数据模式

**显示信息：**
- 当前数据摘要（任务数、完成状态、分类数、最后更新时间）
- 共享状态（IP地址、端口）
- 实时共享状态指示

**操作：**
- 启动共享：开始P2P服务器，监听连接
- 停止共享：停止服务器，断开所有连接

### 接收数据模式

**显示信息：**
- 扫描到的设备列表（设备名称和IP地址）
- 接收到的数据预览
- 数据覆盖警告（如果有现有数据）

**操作：**
- 扫描局域网设备：查找可用设备
- 选择设备：点击设备列表中的设备
- 确认导入：导入接收到的数据
- 取消导入：放弃本次导入

## 安全提示

### 数据覆盖警告

当接收数据时，如果本地已有数据，系统会显示黄色警告框：

```
⚠️ 注意: 导入数据将覆盖当前所有数据，建议先备份数据！
```

**建议操作：**
1. 导入前确认是否真的需要覆盖
2. 系统会自动创建备份（保存在 `data/backups/` 目录）
3. 如需恢复备份，请查看文档 DATA_TRANSFER_FEATURE.md

### 局域网环境

- 该功能仅适用于可信的局域网环境
- 不支持跨网络或互联网传输
- 建议在家庭或办公网络内使用

## 常见问题

### Q: 扫描不到其他设备？

**可能原因：**
1. 设备不在同一局域网
2. 防火墙阻止了端口5000
3. 目标设备未启动共享

**解决方法：**
1. 检查两台设备的网络连接
2. 暂时关闭防火墙或添加例外规则
3. 确认目标设备的共享状态为"正在共享"

### Q: 连接超时或失败？

**可能原因：**
1. IP地址输入错误
2. 目标设备网络中断
3. 防火墙阻止

**解决方法：**
1. 重新扫描获取正确的IP地址
2. 检查目标设备的网络连接
3. 检查防火墙设置

### Q: 导入数据失败？

**可能原因：**
1. 数据格式不正确
2. 数据库文件权限问题
3. 磁盘空间不足

**解决方法：**
1. 确认数据来源可靠
2. 检查data目录的读写权限
3. 清理磁盘空间
4. 查看错误日志获取详细信息

## 最佳实践

### 1. 定期备份数据

在导入新数据前，系统会自动创建备份。但建议定期手动备份重要数据。

备份文件位置：`data/backups/todo_backup_YYYYMMDD_HHMMSS.db`

### 2. 在空闲时间同步

由于数据传输可能需要一些时间，建议在网络不繁忙时进行同步操作。

### 3. 确认数据来源

只从可信的设备接收数据，避免导入恶意或错误的数据。

### 4. 先测试后使用

首次使用时，建议先用少量测试数据验证功能是否正常工作。

## 技术细节

### 通信协议

- **传输协议**: TCP Socket
- **数据格式**: JSON
- **默认端口**: 5000
- **编码格式**: UTF-8

### 数据包结构

每个数据包包含：
1. 长度头（4字节，网络字节序）
2. JSON数据主体

示例：
```
[4字节长度][JSON数据]
```

### 支持的数据类型

- ✅ 任务（包括周期性任务）
- ✅ 分类
- ✅ 应用设置
- ❌ 附件文件（如有）

## 性能说明

### 扫描速度

扫描局域网设备可能需要几秒钟，具体取决于：
- 网络规模
- 网络速度
- 防火墙设置

### 传输速度

数据传输速度取决于：
- 网络带宽
- 数据量大小
- 网络延迟

典型的传输速度：几KB/s到几MB/s

### 内存占用

- 服务器模式：约10-20MB
- 客户端模式：约5-10MB

## 故障排查

### 启用调试日志

如果遇到问题，可以查看应用的日志输出：

```python
# 在启动应用时，日志会输出到控制台
python main.py
```

常见的日志信息：
- `收到来自 192.168.1.x 的连接` - 表示有设备尝试连接
- `接收到来自 xxx 的数据` - 表示数据接收成功
- `数据导出成功` - 表示数据准备完成

### 重置功能

如果遇到问题，可以尝试：
1. 停止共享并重新启动
2. 关闭应用并重新打开
3. 重启网络连接
