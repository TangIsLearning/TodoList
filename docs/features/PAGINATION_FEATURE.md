# 分页功能说明

## 功能概述

任务列表现已支持分页显示，通过后端数据库查询实现，提升大数据量下的性能和用户体验。

## 支持的查询条件

### 1. 分类筛选
- **全部**: 显示所有分类的任务
- **未分类**: 显示没有分类的任务
- **具体分类**: 显示指定分类下的任务

### 2. 优先级筛选
- **所有优先级**: 不限制优先级
- **高优先级** (🔴): 只显示高优先级任务
- **中优先级** (🟡): 只显示中优先级任务
- **低优先级** (🟢): 只显示低优先级任务
- **无优先级** (⚪): 只显示无优先级任务

### 3. 状态筛选
- **所有状态**: 不限制状态
- **已完成**: 只显示已完成的任务
- **未完成**: 只显示未完成的任务
- **未完成未逾期**: 只显示未完成且未超过截止日期的任务
- **未完成已逾期**: 只显示未完成且已超过截止日期的任务

### 4. 时间筛选
- **所有时间**: 不限制时间
- **今天**: 显示截止日期为今天的任务
- **明天**: 显示截止日期为明天的任务
- **本周内**: 显示截止日期在本周内的任务（7天内）
- **本月内**: 显示截止日期在本月内的任务
- **无截止时间**: 显示没有设置截止日期的任务

### 5. 自定义日期筛选
- 通过日历视图点击具体日期，可以查看该日期的任务
- 自定义日期筛选会覆盖时间筛选条件

### 6. 搜索关键词
- 支持搜索任务标题和描述
- 搜索时会在所有筛选条件的基础上进一步过滤

### 7. 年月筛选（预留接口）
- 支持按年份筛选
- 支持按月份筛选
- 可用于月度报表等功能

## 分页功能

### 单页数量配置
支持以下单页显示数量选项：
- **10条/页**（默认）
- **20条/页**
- **50条/页**
- **100条/页**

### 分页导航
- **首页**: 跳转到第一页
- **上一页**: 返回上一页
- **页码**: 直接点击页码跳转
- **下一页**: 跳转到下一页
- **末页**: 跳转到最后一页

### 智能页码显示
- 当总页数 ≤ 5 时，显示所有页码
- 当总页数 > 5 时，智能显示相关页码：
  - 前几页：显示 1-4 和最后一页
  - 后几页：显示第一页和后4页
  - 中间页：显示第一页、当前页前1页、当前页、当前页后1页、最后一页

## 使用说明

### 基本使用
1. 任务列表底部显示分页控件
2. 查看当前显示范围和总数
3. 通过下拉菜单调整每页显示数量
4. 使用导航按钮切换页面

### 组合筛选
所有筛选条件可以组合使用，例如：
- 查看"工作"分类下"高优先级"且"未完成"的任务
- 查看"本周内"所有"已完成"的任务

### 筛选条件重置
当切换以下条件时，会自动重置分页到第一页：
- 切换分类
- 切换优先级
- 切换状态
- 切换时间筛选
- 修改搜索关键词

### 排序规则
任务列表默认按以下规则排序：
1. **优先级**：高 > 中 > 低 > 无
2. **截止日期**：截止日期早的排在前面
3. **创建时间**：新创建的排在后面

## 技术实现

### 后端实现
- **数据库查询**: 使用SQLite的`LIMIT`和`OFFSET`实现分页
- **条件筛选**: 使用SQL的`WHERE`子句实现多条件筛选
- **排序**: 使用`ORDER BY`子句实现复合排序

### 前端实现
- **API调用**: 通过`pywebview.api.get_todos()`调用后端分页接口
- **UI组件**: 使用Flexbox布局实现响应式分页组件
- **事件处理**: 支持点击、键盘快捷键等多种交互方式

## 性能优化

### 数据库层面
- 分页查询避免一次性加载所有数据
- 使用索引优化查询性能（建议在常用字段上创建索引）

### 前端层面
- 只渲染当前页的任务数据
- 使用事件委托减少事件监听器数量
- 智能页码显示减少DOM节点数量

## 注意事项

1. **筛选条件优先级**: 自定义日期筛选 > 时间筛选 > 其他筛选
2. **分页重置**: 任何筛选条件改变都会重置到第一页
3. **数据一致性**: 新增/删除任务后会自动重新加载当前页
4. **移动端适配**: 分页控件在移动端会自动调整布局
