# 周期性任务功能实现说明

## 功能概述

本次更新为TodoList应用新增了完整的周期性任务管理功能，包括：

1. **创建周期性任务**：支持按年、按月、按周重复
2. **循环次数设置**：支持设置具体的循环次数或无限循环
3. **智能删除**：支持删除单个任务或整个周期的任务
4. **任务标识**：在界面上清晰标识周期性任务

## 技术实现

### 后端改进

#### 1. 数据库模型扩展
- **新增字段**：
  - `is_recurring`: 是否为周期性任务
  - `recurrence_type`: 重复周期类型（yearly/monthly/weekly）
  - `recurrence_interval`: 重复间隔（默认为1）
  - `recurrence_count`: 循环次数（None表示无限循环）
  - `parent_task_id`: 父任务ID（用于关联周期性子任务）

#### 2. 数据库迁移
- 自动检测并添加新字段，确保向后兼容
- 使用`PRAGMA table_info`获取现有表结构
- 通过`ALTER TABLE`语句动态添加新字段

#### 3. 周期性任务生成算法
- 使用`python-dateutil`库进行日期计算
- 支持按年、月、周的精确日期递推
- 自动创建父任务和所有子任务
- 遵循循环次数限制

#### 4. API接口扩展
- `add_recurring_todo`: 创建周期性任务系列
- `delete_todo`: 增强支持删除单个或整个周期
- 参数验证和错误处理

### 前端改进

#### 1. 用户界面增强
- 在"更多选项"中添加周期性任务设置
- 包含复选框、周期类型选择、循环次数输入
- 响应式设计，移动端友好

#### 2. 样式优化
- 新增`.recurring-options`样式
- 周期性任务标识徽章
- 删除选项对话框的美化
- 完整的暗色主题支持

#### 3. 交互逻辑
- 周期性任务禁止编辑（保护数据一致性）
- 智能删除对话框（单个/整个周期选择）
- 成功消息的详细反馈

#### 4. 任务渲染
- 为周期性任务添加特殊标识
- 父任务显示"周期性"徽章
- 子任务显示"周期任务"徽章

## 使用方法

### 创建周期性任务

1. 点击"添加任务"按钮
2. 填写基本任务信息
3. 点击"更多选项"展开高级设置
4. 勾选"创建为周期性任务"
5. 选择重复周期：
   - **按周**：每周重复一次
   - **按月**：每月重复一次
   - **按年**：每年重复一次
6. 设置循环次数（可选，留空为无限循环）
7. **必须设置截止时间**（作为周期的起始时间）
8. 点击"保存"

### 删除周期性任务

1. 点击任务旁边的删除按钮
2. 系统会自动检测是否为周期性任务
3. 选择删除方式：
   - **仅删除此任务**：只删除当前选中的任务
   - **删除整个周期**：删除该周期的所有任务
4. 确认删除

### 注意事项

- **编辑限制**：周期性任务不支持编辑，如需修改请删除后重新创建
- **截止时间**：周期性任务必须设置截止时间，作为重复计算的基准
- **数据一致性**：系统会自动维护父子任务关系，确保数据完整性
