# 截止日期默认值优化总结

## 问题概述
用户反馈新建任务时存在两个问题：
1. 默认会选定截止日期为明天
2. 页面渲染效果不够友好，显示为"yyyy/MM/日 --：--"

## 解决方案

### 1. 移除默认截止日期设置
- 将新建任务时的截止日期默认值从"明天"改为"空"
- 让用户根据实际需要自行设置截止日期

### 2. 优化用户界面显示
- 添加友好的占位符提示
- 在标签中明确标注截止日期是可选的
- 改善空值时的视觉效果

## 修改的文件

### `frontend/js/todo.js`

**修改方法：** `showAddTaskModal()`

**修改前：**
```javascript
// 设置默认截止时间为明天同一时间（本地时间）
const tomorrow = new Date();
tomorrow.setDate(tomorrow.getDate() + 1);
const defaultDueDate = this.toDateTimeLocalString(tomorrow);
document.getElementById('task-due-date').value = defaultDueDate;
```

**修改后：**
```javascript
// 截止日期默认为空，不设置默认值
document.getElementById('task-due-date').value = '';
```

### `frontend/index.html`

**修改内容：** 截止日期输入框标签和属性

**修改前：**
```html
<div class="form-group">
    <label for="task-due-date">截止日期</label>
    <input type="datetime-local" id="task-due-date" value="">
</div>
```

**修改后：**
```html
<div class="form-group">
    <label for="task-due-date">截止日期 <span style="color: #666; font-size: 12px;">（可选）</span></label>
    <input type="datetime-local" id="task-due-date" value="" placeholder="选择截止日期和时间">
</div>
```

### `frontend/css/components.css`

**新增样式：**
```css
/* 日期时间输入框特殊样式 */
.form-group input[type="datetime-local"] {
    min-height: 40px;
}

.form-group input[type="datetime-local"]:invalid:not(:focus):placeholder-shown {
    color: var(--text-muted);
}
```

**移动端适配：**
```css
.form-group input[type="datetime-local"] {
    font-size: var(--font-size-sm);
}
```
