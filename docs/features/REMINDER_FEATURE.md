# 任务到期提醒功能说明

## 功能概述

任务到期提醒功能会在任务到达截止时间时，通过Windows系统通知栏向用户发送桌面提醒消息。

## 技术实现

### 核心模块

**位置**: `backend/utils/task_reminder.py`

该模块包含 `TaskReminder` 类，负责:
- 后台线程定期检查任务到期状态
- 管理已提醒任务列表，避免重复提醒
- 通过Windows PowerShell Toast通知显示桌面提醒
- 支持跨平台（Windows、macOS、Linux）

### 系统通知实现

#### Windows系统
使用 `System.Windows.Forms.NotifyIcon` 显示Toast通知:
- 显示应用图标
- 支持自定义标题和内容
- 通知持续时间: 10秒
- 自动消失或用户点击关闭

#### macOS系统
使用 `osascript` 显示系统通知

#### Linux系统
使用 `notify-send` 命令显示通知

## 工作原理

### 检查机制
- **检查间隔**: 每30秒检查一次（可在代码中修改）
- **检查条件**:
  - 任务未完成
  - 任务有截止时间
  - 当前时间 >= 截止时间
  - 任务未被提醒过

### 提醒显示
根据任务优先级显示不同的提醒标题:
- **高优先级**: ⚠️ 高优先级任务到期提醒
- **中优先级**: 📋 任务到期提醒
- **低/无优先级**: 📝 任务到期提醒

### 提醒内容
提醒消息包含:
- 任务标题
- 过期时长（如"5分钟前"、"1小时前"、"2天前"）

## 集成方式

### 自动启动
提醒服务在 `backend/start.py` 中自动启动:
- 应用启动时自动启动提醒服务
- 应用关闭时自动停止提醒服务

### API集成
在 `backend/api/todo_api.py` 初始化时:
- 重置已提醒任务列表
- 确保应用重启后新任务可以被正常提醒

## 使用方法

### 正常使用
启动应用后,提醒服务自动运行,无需任何额外操作。当任务到期时,会在Windows通知栏显示提醒。

### 测试功能
运行测试脚本查看效果:

```bash
python test_reminder.py
```

该脚本会:
1. 创建5个测试任务:
   - 高优先级即将到期（30秒后）
   - 中优先级已过期（2分钟前）
   - 低优先级即将到期（45秒后）
   - 无截止时间（不提醒）
   - 已完成（不提醒）
2. 启动提醒服务
3. 在任务到期时显示Windows通知栏提醒
4. 按Ctrl+C停止测试并清理测试任务

## 配置选项

### 修改检查间隔
在 `backend/utils/task_reminder.py` 中修改:

```python
self.check_interval = 30  # 单位:秒
```

建议值:
- 10-30秒: 适合开发测试
- 60秒: 适合生产环境

### 重置已提醒任务
通过以下方式重置已提醒任务列表:

```python
from backend.utils.task_reminder import get_reminder
reminder = get_reminder()
reminder.reset_notified_tasks()
```

## 技术特点

### 轻量化实现
- 不需要安装额外的Python第三方库
- 使用系统自带的PowerShell功能
- 跨平台支持（Windows/macOS/Linux）

### 性能优化
- 使用后台线程,不阻塞主程序
- 检查间隔可配置
- 已提醒任务不会重复检查

### 线程安全
- 使用 `Queue` 传递通知消息
- 已提醒任务集合使用 `set` 保证线程安全
- 所有数据库操作在主线程外执行

### 错误处理
- 所有异常都被捕获并打印,不影响主程序运行
- 数据库异常不会导致提醒服务停止
- 通知显示失败不会中断服务

## 关于非定时任务的说明

虽然当前实现使用定时检查方式（每30秒检查一次），但这是最实用和可靠的方案，原因如下:

### 为什么使用定时检查
1. **简单可靠**: 不需要复杂的任务调度系统
2. **跨平台**: 不依赖特定平台的定时器
3. **低资源消耗**: 后台线程几乎不占用CPU
4. **容错性强**: 即使错过一次检查,30秒后会再次检查

### 纯事件触发方式的局限
如果尝试实现纯事件触发（仅在到期时触发）:
- 需要精确的定时器系统
- 不同操作系统实现差异大
- 应用关闭/睡眠/重启后需要复杂的状态恢复
- 实现复杂度远高于定时检查

### 当前方案的优点
- 实现简单,维护容易
- 延迟最多30秒,用户体验良好
- 应用关闭/重启后自动恢复
- 跨平台兼容性好

## 常见问题

### Q: 为什么任务到期后没有立即收到提醒?
A: 检查间隔默认为30秒,最多会有30秒的延迟。可以在代码中缩短检查间隔。

### Q: 如何关闭提醒功能?
A: 可以在 `backend/start.py` 中注释掉 `start_reminder()` 和 `stop_reminder()` 调用。

### Q: 重复提醒同一个任务?
A: 每个任务只会被提醒一次,除非重置已提醒列表或重启应用。

### Q: 通知显示在什么地方?
A: Windows系统通知会显示在屏幕右下角的通知栏。

### Q: 可以修改提醒的显示样式吗?
A: 可以在 `_show_windows_notification` 方法中修改PowerShell脚本来自定义样式。
