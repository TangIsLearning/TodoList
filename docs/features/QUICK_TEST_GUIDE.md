## 完整测试（两台设备）

### 设备A操作

1. **启动应用**
   ```bash
   python main.py
   ```

2. **启动数据共享**
   - 点击左侧"🔗 数据共享"按钮
   - 选择"共享数据"标签页
   - 点击"启动共享"
   - 记录显示的IP（例如：`192.168.1.100:5000`）

3. **确认共享状态**
   - 应该看到"共享状态"区域
   - 显示IP地址和端口
   - 有一个绿色的脉冲圆点

### 设备B操作

1. **启动应用**
   ```bash
   python main.py
   ```

2. **扫描设备**
   - 点击左侧"🔗 数据共享"按钮
   - 选择"接收数据"标签页
   - 点击"扫描局域网设备"
   - 等待扫描完成（约3-10秒）

3. **查看扫描结果**
   - 应该看到"可用设备"列表
   - 显示设备A的IP地址：`192.168.1.100`

4. **测试数据传输**
   - 点击设备A的IP
   - 等待接收数据
   - 查看数据预览
   - （可选）确认导入

## 常见问题速查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 扫描超时 | 网络问题 | 检查网络连接 |
| 未发现设备 | 对方未启动 | 确认设备A已启动共享 |
| 防火墙阻止 | 端口被阻止 | 临时关闭防火墙 |
| IP地址不对 | 网络配置 | 检查两台设备在同一网段 |
| 连接被拒 | 对方拒绝连接 | 重启应用重试 |

## 网络测试命令

### Windows

```cmd
# 测试网络连通性
ping 192.168.1.100

# 测试端口连通性
telnet 192.168.1.100 5000
# 或
Test-NetConnection -ComputerName 192.168.1.100 -Port 5000
```

### Linux/Mac

```bash
# 测试网络连通性
ping 192.168.1.100

# 测试端口连通性
telnet 192.168.1.100 5000
# 或
nc -zv 192.168.1.100 5000
```

## 防火墙配置

### Windows防火墙

```cmd
# 添加防火墙规则
netsh advfirewall firewall add rule name="TodoList P2P" dir=in action=allow protocol=TCP localport=5000

# 删除规则
netsh advfirewall firewall delete rule name="TodoList P2P"
```

### Linux (ufw)

```bash
# 允许端口
sudo ufw allow 5000/tcp

# 查看状态
sudo ufw status
```

### macOS

```bash
# 添加防火墙规则（系统偏好设置 -> 安全性与隐私 -> 防火墙）
# 或使用命令行
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /path/to/python
```
