# 数据传输功能说明

## 功能概述

数据传输功能允许用户在局域网内的多台设备之间共享和同步TodoList数据。该功能基于Python P2P实现，支持数据的导出、导入和传输。

## 主要特性

1. **P2P数据传输** - 基于Socket实现的局域网P2P数据传输
2. **数据序列化** - 自动处理SQLite数据库的序列化和反序列化
3. **设备发现** - 自动扫描局域网内的可用设备
4. **数据备份** - 导入数据前自动备份现有数据
5. **安全提示** - 在数据覆盖前提示用户确认

## 使用方法

### 共享数据（发送方）

1. 点击左侧导航栏左下角的"🔗 数据共享"按钮
2. 选择"共享数据"模式
3. 查看当前数据摘要，确认要共享的数据
4. 点击"启动共享"按钮
5. 系统会显示本机IP地址和端口
6. 其他设备可以通过该IP地址接收数据

### 接收数据（接收方）

1. 点击左侧导航栏左下角的"🔗 数据共享"按钮
2. 选择"接收数据"模式
3. 点击"扫描局域网设备"按钮
4. 从设备列表中选择要连接的设备
5. 查看接收到的数据预览
6. 确认后点击"确认导入"导入数据
7. **重要提示**: 导入数据将覆盖当前所有数据，系统会提示备份

## 技术实现

### 后端组件

#### 1. P2P服务器 (`backend/p2p/p2p_server.py`)
- 监听指定端口（默认5000）
- 接受来自局域网设备的连接
- 处理数据传输请求
- 使用多线程处理多个客户端连接

#### 2. P2P客户端 (`backend/p2p/p2p_client.py`)
- 扫描局域网内的可用设备
- 向指定设备发送数据请求
- 接收和解析传输的数据

#### 3. 数据管理器 (`backend/p2p/data_manager.py`)
- 导出SQLite数据库中的所有数据
- 导入数据到数据库
- 在导入前创建备份
- 提供数据摘要信息

#### 4. API接口 (`backend/api/todo_api.py`)
新增API方法：
- `p2p_start_server()` - 启动P2P服务器
- `p2p_stop_server()` - 停止P2P服务器
- `p2p_scan_devices()` - 扫描设备
- `p2p_receive_data(ip)` - 从指定IP接收数据
- `p2p_export_data()` - 导出当前数据
- `p2p_get_data_summary()` - 获取数据摘要
- `p2p_has_data()` - 检查是否有数据
- `p2p_import_data(data, backup)` - 导入数据

### 前端组件

#### 1. HTML结构 (`frontend/index.html`)
- 数据传输模态框
- 共享模式和接收模式面板
- 设备列表显示
- 数据预览和确认界面

#### 2. CSS样式 (`frontend/css/main.css`)
- 模态框样式
- 模式切换按钮
- 设备列表样式
- 响应式设计支持

#### 3. JavaScript逻辑 (`frontend/js/dataTransfer.js`)
- DataTransfer类管理数据传输流程
- 与后端API通信
- UI交互逻辑
- 数据验证和确认

## 安全注意事项

1. **局域网使用** - 该功能仅设计用于可信的局域网环境
2. **数据覆盖** - 导入数据会完全覆盖现有数据，请谨慎操作
3. **自动备份** - 系统会在导入前自动创建备份，备份文件保存在 `data/backups/` 目录
4. **防火墙** - 确保防火墙允许应用的端口（默认5000）通信

## 故障排除

### 无法扫描到设备
- 确保设备在同一局域网
- 检查防火墙设置
- 确认目标设备已启动共享

### 连接失败
- 检查IP地址是否正确
- 确认目标设备的应用正在运行
- 检查网络连接

### 导入失败
- 确认接收到的数据格式正确
- 检查数据库文件权限
- 查看错误日志
