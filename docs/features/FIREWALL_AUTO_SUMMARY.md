## 🔍 技术细节

### 防火墙规则规格
- **规则名称**: `TodoList P2P Server`
- **方向**: 入站 (in)
- **操作**: 允许 (allow)
- **协议**: TCP
- **端口**: 5353（可配置）
- **配置文件**: 任意 (any)
- **描述**: `TodoList P2P数据传输服务`

### 执行流程

#### 启动P2P共享
```
1. 用户点击"开始共享"
2. 前端调用 pywebview.api.p2p_start_server()
3. API层导出数据
4. API层创建P2PServer实例
5. Server自动初始化FirewallManager
6. FirewallManager.add_rule() 添加防火墙规则
7. P2PServer.start() 启动服务器
8. 返回成功信息和防火墙配置状态
9. 前端显示详细提示
```

#### 停止P2P共享
```
1. 用户点击"停止共享"
2. 前端调用 pywebview.api.p2p_stop_server()
3. P2PServer.stop() 停止服务器
4. FirewallManager.remove_rule() 删除防火墙规则
5. 返回成功信息
6. 前端显示提示
```

### 权限处理

#### 有管理员权限
```
✓ 自动添加防火墙规则成功
✓ 服务器正常启动
✓ 其他设备可以连接
✓ 停止时自动删除规则
```

#### 无管理员权限
```
✗ 防火墙规则添加失败
⚠ 显示警告信息
✓ 服务器仍然启动成功
⚠ 其他设备可能无法连接
✓ 停止时不会尝试删除规则
```

### 异常处理

#### 防火墙添加失败
```
1. 捕获异常
2. 记录详细日志
3. 不影响服务器启动
4. 向用户显示警告信息
```

#### 服务器启动失败
```
1. 捕获异常
2. 记录详细日志
3. 自动清理已添加的防火墙规则
4. 返回失败信息
```

#### 防火墙删除失败
```
1. 捕获异常
2. 记录详细日志
3. 不影响服务器停止
4. 向用户显示警告信息
```

## 🔒 安全性

### 自动清理机制
- 停止共享时自动删除规则
- 服务器启动失败时自动清理
- 避免留下永久的安全漏洞

### 最小权限原则
- 只开放必要的端口（5353）
- 只允许TCP协议
- 只在共享期间开放

### 透明性
- 规则命名清晰明确
- 用户可以随时查看规则
- 提供手动管理方法

## 💡 使用建议

### 推荐做法
1. **以管理员身份运行** - 完全自动，无需手动配置
2. **验证配置** - 首次使用后验证防火墙规则
3. **测试连接** - 在实际网络环境中测试
4. **定期检查** - 检查防火墙规则是否正确

1. ❌ 长期保持防火墙规则（应用会自动管理）
2. ❌ 手动修改自动创建的规则
3. ❌ 在不信任的网络中使用
4. ❌ 关闭系统防火墙
